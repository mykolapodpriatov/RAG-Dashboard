# План реализации проекта RAG-Dashboard

## 1. Идея и описание
**RAG-Dashboard** – визуальная панель для оценки и сравнения RAG-потоков.
Проект объединяет возможности существующих инструментов оценки (Open RAG Eval, Ragas) в удобный интерактивный веб-интерфейс. Предназначен для загрузки результатов различных экспериментов, сравнения метрик и визуализации динамики качества RAG-систем.

## 2. Уникальные особенности
- **Единый графический интерфейс:** легкая загрузка файлов (CSV/JSON) с результатами тестов и автоматическое построение дашбордов.
- **Поддержка стандартизированных метрик:** нативная интеграция метрик, не требующих «золотых» ответов (UMBRELA, AutoNuggetizer), а также классических из Ragas.
- **Пользовательские метрики:** возможность легко расширять список метрик и строить по ним графики.
- **Интеграция с RAG (Live Evaluation):** панель может запускать процесс оценки пользовательской RAG напрямую из интерфейса.

## 3. Выбор технологий
- **Фронтенд и Бэкенд:** Streamlit (для быстрого старта) или FastAPI + React.
- **Оценка:** Open RAG Eval, Ragas.
- **Визуализация:** Plotly, Pandas для обработки датафреймов.

## 4. Поэтапный план разработки

### Этап 1. Базовый интерфейс и загрузка данных
- Создать веб-панель с функцией загрузки файлов с историей RAG-запросов и ответов.
- Реализовать парсинг данных и вывод таблиц.

### Этап 2. Интеграция с Ragas и Open RAG Eval
- Написать обертки (wrappers) для вызова оценки качества генерации и поиска прямо из приложения по загруженным данным.

### Этап 3. Визуализация и аналитика
- Добавить графики распределения метрик (faithfulness, answer_relevancy, context_precision).
- Реализовать функционал сравнения двух и более запусков (экспериментов).

### Этап 4. Тестирование и оптимизация
- Подготовить несколько демо-датасетов для показа работы дашборда "из коробки".
- Протестировать производительность при больших объемах данных.

### Этап 5. Документация и публикация
- Написать инструкции по развертыванию, описать поддерживаемые форматы данных.
- Создать Dockerfile.
- Опубликовать на GitHub.
